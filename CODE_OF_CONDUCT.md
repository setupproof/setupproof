# Code Of Conduct

SetupProof is a technical project. Project spaces should stay useful, direct,
and respectful.

## Scope

This code applies in repository issues, pull requests, discussions, reviews,
and project-affiliated community spaces.

## Standards

Participants are expected to:

- focus criticism on ideas, code, docs, and behavior;
- give enough context for others to reproduce or understand a concern;
- respect maintainer decisions about project scope, release timing, and safety;
- avoid sharing secrets, private hostnames, or private repository details in
  public threads;
- make room for contributors with different experience levels.

Unacceptable behavior includes:

- harassment, threats, or personal attacks;
- discriminatory language or conduct;
- repeated off-topic disruption after a maintainer asks for focus;
- publishing another person's private information without consent;
- using project spaces to pressure maintainers for private support, security
  details, or undisclosed release plans.

## Reporting

Report conduct concerns privately to the maintainers through a private channel
listed by the SetupProof organization or repository. If no dedicated contact is
available yet, contact a maintainer directly rather than opening a public issue.

Include the relevant links, screenshots, dates, and any immediate safety
concerns. Do not include unrelated private information.

## Enforcement

Maintainers may remove comments, edit or close threads, warn participants,
limit participation, block accounts, or report abuse to the hosting platform.
Enforcement should be proportionate to the behavior and focused on protecting
the project community.
